                          KEY FINDINGS AT A GLANCE
================================================================================

GOLD STANDARD CONFIG:
  • Oh My Tmux! (gpakosz/.tmux) — Actively maintained 2025, 100+ contributors
  • Philosophy: Never edit core config; customize via .tmux.conf.local
  • Dual prefix (C-a + C-b), SSH-aware, Powerline theming

ESSENTIAL PLUGINS (TPM):
  1. tmux-sensible       — Sensible defaults
  2. vim-tmux-navigator  — Seamless Vim ↔ tmux navigation
  3. tmux-yank           — System clipboard integration
  4. tmux-resurrect      — Session persistence across restarts
  5. tmux-continuum      — Auto-save every 5 minutes

CRITICAL SETTINGS FOR AI DEVELOPMENT:
  set -g escape-time 0                 # Neovim responsiveness
  set -g history-limit 250000          # Claude Code output
  set -g allow-passthrough on          # DEC 2026 synchronized output
  set -g extended-keys on              # Alt key combinations
  set -g focus-events on               # TUI app focus tracking
  set -g set-clipboard on              # OSC 52 native clipboard
  set -g status-interval 10            # Reduce CPU usage

INTEGRATION PATTERNS:
  Pattern 1: Neovim + Claude in same session (windows + panes)
  Pattern 2: Multi-agent autonomous dev (one agent per window)
  Pattern 3: Floating popups (Ctrl+G, Ctrl+S, Ctrl+J, Ctrl+F)

PERFORMANCE CHARACTERISTICS:
  • Memory: 2 MB (idle) → 45-60 MB (4 panes + 250K scrollback)
  • CPU: <0.1% idle → 0.5-2% active
  • Latency: <5ms keyboard, <2ms pane switch, <10ms status update

BEST PRACTICES SUMMARY:
  ✓ Never edit core config — use .tmux.conf.local
  ✓ Keep status line minimal — session name + time
  ✓ Use vi-style copy-mode with system clipboard
  ✓ Disable mouse for AI workflows (Shift+drag when needed)
  ✓ Set escape-time to 0 for Neovim
  ✓ Use hooks for automation (after-new-session, pane-exited)
  ✓ Leverage git worktrees for multi-agent isolation
  ✓ Use display-popup for floating tools (3.2+)

NEW TOOLS (2025-2026):
  • dmux (Rust) — Workspace manager for multi-agent setups
  • sessionx — FZF session switcher with preview
  • sesh — Intelligent session detection
  • treemux (Rust) — File explorer sidebar
  • laio (Rust) — Flexbox-inspired layouts

VERSION COMPATIBILITY:
  • Minimum: tmux 3.0+ (acceptable)
  • Recommended: tmux 3.2+ (floating popups, OSC 52)
  • Legacy: 2.9 (most features work, missing advanced features)

                              QUICK TIPS
================================================================================

Installation:
  macOS:        brew install tmux
  Ubuntu:       sudo apt-get install tmux
  From source:  git clone & ./configure && make

Oh My Tmux Setup (3 lines):
  git clone https://github.com/gpakosz/.tmux.git ~/.tmux
  ln -s -f ~/.tmux/.tmux.conf ~/.tmux.conf
  cp ~/.tmux/.tmux.conf.local ~/.tmux.conf.local

Essential Keybindings:
  C-a d       Detach session
  C-a c       New window
  C-h/j/k/l   Navigate panes (vim-tmux-navigator)
  C-a [       Enter copy-mode
  C-a r       Reload config

Copy-Paste System:
  v           Begin selection
  y           Copy to buffer
  Y           Copy line + system clipboard
  Shift+Drag  Select across panes (bypasses tmux)

Reload Config:
  C-a r       Or: tmux source-file ~/.tmux.conf

Check Version:
  tmux -V     (Output: tmux 3.2a, etc.)

================================================================================
                            KEY TAKEAWAYS
================================================================================

1. Oh My Tmux! is the best-in-class starting point for 2025-2026. The config
   separation pattern (core vs. local) makes upgrades painless.

2. For AI-assisted development (Claude Code, Neovim), critical settings are:
   escape-time=0, history-limit=250K, allow-passthrough=on, extended-keys=on,
   focus-events=on, set-clipboard=on, status-interval=10s.

3. vim-tmux-navigator enables seamless keyboard navigation between Vim panes
   and tmux windows without the prefix key.

4. Multi-agent development patterns pair tmux windows with git worktrees for
   complete isolation and parallel development without merge conflicts.

5. Floating popups (display-popup in tmux 3.2+) provide quick access to tools
   (fzf, git status) without reshaping the main layout.

6. Hooks (after-new-session, pane-exited) enable automation of setup and
   teardown tasks, perfect for Shipwright daemon integration.

7. Mouse mode introduces trade-offs. For AI tools, keep mouse OFF and use
   Shift+drag when cross-pane selection is needed.

8. Status bar should be minimal (session name, window list, time) with updates
   every 5-10 seconds to reduce CPU while staying informative.

9. The plugin ecosystem is mature: TPM for management, with essential plugins
   for navigation, persistence, and clipboard. Modern tools (dmux, sessionx)
   add specialized features for advanced workflows.

10. Performance is excellent: tmux overhead is <1% CPU, with most time spent
    in terminal rendering. Memory scales linearly with scrollback (250K
    history = ~45 MB for 4-pane session).

For Shipwright users and AI-assisted development teams in 2025-2026:
